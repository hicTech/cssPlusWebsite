Intro

Css Plus want to provide a simple and fast way to realize flexible layout based on flexbox modeling. This awesome css feature lets you specify a flexible container within which direct children can be also flexible and float as columns or rows.

You can read a complete guide to flexbox here:
http://css-tricks.com/snippets/css/a-guide-to-flexbox/

Starting from these specifics cssPlus is just a collection of css classes that allows you to create complex and flexible layouts in seconds based on these main concepts:

1. flexibility and/or extensiveness
2. table approach (rows and cols)
3. alignments
4. overflow delegation


Flexibility and/or extensiveness: this means that we want to count on specific elements (a set of elements or a container) that can stretch or shrink its sizes to fit all the available space and we want this behavior in both horizontal and vertical direction.

Table approach: all we have to do is thinking about layout with dear old table approach (with rows and columns). You can easy check that allmost of all components of a web applications can be realized with rows and cols.

Alignments: we also want to have a robust way to set elements alignments: top, middle and bottom for vertical elements and left, right and center for horizontals

Overflow delegation: this means that we don't have to think about overflow while we are coding primary layout. This must be our structure scaffolding that defines main spaces and how they have to follow window/container sizes variations. This scaffolding will contain our components (menu, buttons, text, media object,...) and we have to manage these components overflow, one by one and each independently.

Introducing cssPlus
It all start from an .EXTENDER (class="EXTENDER"). The .EXTENDER is the root/main container that ensures us the first main concept (flexibility and/or extensiveness). This is a "position:absolute" element that forces its sizes to fit all the available space of its container. When we use the term "container" we refer to browser viewport or to a generic container nested in some place inside our interface. The .EXTENDER gives to its content all the space that it can reach (with no border nor margins nor paddings) inside its container, if we want to leave a frame all around (like a padding) we can use .PADDER (.PADDER5, .PADDER10, .PADDER15,...).

For example .PADDER5 will fit all the available space inside an .EXTENDER leaving a 5 pixel frame all around.

The .EXTENDER is a mandatory container which can optionally contains a .PADDER.
A .PADDER can't stands alone without an .EXTENDER as direct father.

Let's introduce .ROWS and .COLS: after an .EXTENDER/.PADDER we can nest only a .ROWS or a COLS. .ROWS and .COLS implements, in a crossbrowser way, a flexible container (display: flex;) of flexible elements that can be rows or columns.

.ROWS / .ROW / .ROW-FLEX
A. ROWS element is a flexible component that enables a flex context for all its direct children. These children will be 100% width rows with static height (' class="ROW" style="height:50px" ') or dynamic hieght (' "ROW-FLEX" ')

.COLS / .COL / .COL-FLEX
A. COLS elements is a flexible component that enables a flex context for all its direct children. These children will be 100% height columns with static width (' "COL" style="width:50px" ') or dynamic width (' "COL-FLEX" ')

The main concept behind .COLS and .ROWS is: starting from the superior container, no matter if is a flexible or a static one, the .ROWS is it self a flexible container whose children specify just their heights with width:100%. The .COLS is it self a flexible container whose children specify just their widths with height:100%.
Then if we want:
- a static .COL we use class="COL" style="width:50px";
- a flexible .COL we use class="COL-FLEX";
- a static .ROW we can use class="ROW" style="height:50px";
- a flexible .ROW we can use class="ROW-FLEX".
